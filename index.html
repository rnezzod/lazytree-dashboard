📄論文概要

LazyTree: 遅延評価とコンテンツハッシュキャッシュによる長文書RAGの効率化

長文書に対するRAGでは、フラットなチャンク検索ではグローバルな文脈が失われる一方、全文書の要約ツリーを事前構築する手法（RAPTOR等）は初期コストが高く、文書更新に対応できない。

本研究では、セグメント木の遅延伝播アイデアを要約インデックスに応用したLazyTreeを提案する。内部ノードは葉テキストから直接LLMで要約生成（1ノード=1コール）、クエリ到達時にのみ計算される（遅延評価）。

QuALITY 30件評価結果: LazyTree 73.3% > FlatRAG 70.0% > RAPTOR 60.0%
RAPTORの精度低下は「要約の要約」による情報圧縮損失が原因。LazyTreeは葉テキスト直接アクセスでこれを回避。

💡核心アイデア

遅延評価 + ダイレクト要約

必要なノードのみオンデマンド計算。かつ各ノードは葉テキストから直接要約、「要約の要約」による情報損失がない。SHA256ハッシュでセッション横断キャッシュ。

インクリメンタル更新

チャンク更新 → O(log n)根の経路のdirtyマーク → 次回クエリ時のみ再計算。RAPTORは全ツリー再構築が必要。

ハイブリッド検索

内部ノード（大局的文脈）と葉ノード（局所的事実）を同時に提供。これによりFlatRAG（葉のみ）とRAPTOR（内部ノードのみ）の両方を上回る精度を達成。

def _resolve(node):
    # 葉：そのまま返す（LLM不要）
    if node.is_leaf: return node.text
    # キャッシュヒット：0コール
    if cached := cache.get(node.content_hash): return cached
    # ミス：葉テキスト直接（1コール）
    leaves = collect_leaves(node)  # 再帰なし！
    summary = llm_summarize(leaves)
    cache.set(node.content_hash, summary)
    return summary

def query(q, k_int=3, k_leaf=2):
    # Phase 1: スコアリング（LLM不要）
    scores = [sim(proxy(node), q) for node in tree]
    top_internal = top_k(scores, k_int, internal_only=True)
    top_leaves   = top_k(scores, k_leaf, leaves_only=True)
    # Phase 2: 選択ノードのみ解決
    return [_resolve(n) for n in top_internal + top_leaves]
