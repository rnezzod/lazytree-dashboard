📄論文概要

LazyTree: 遅延評価とコンテンツハッシュキャッシュによる長文書RAGの効率化

長文書に対するRAG（Retrieval-Augmented Generation）では、フラットなチャンク検索ではグローバルな文脈が失われる一方、 全文書の要約ツリーを事前構築する手法（RAPTOR等）は初期コストが高く、文書更新に対応できない。

本研究では、セグメント木の遅延伝播アイデアを要約インデックスに応用した LazyTreeを提案する。葉ノードには原文チャンクを格納し、 内部ノードはサブツリー内の葉テキストから直接LLMで要約を生成する（1ノード = 1 LLMコール）。 要約はクエリが到達した時点ではじめて計算され（遅延評価）、 コンテンツハッシュをキーとして永続キャッシュに保存される。 文書更新時はO(log n)の祖先ノードをdirtyマークするのみで即時対応可能。

実験では、RAPTORが31 LLMコールで全ツリーを事前構築するのに対し、 LazyTreeは同等の精度を3コール（90%削減）で達成した。

💡核心アイデア

セグメント木の類比

セグメント木の「遅延伝播」と同様に、必要になるまで要約を計算しない。一度計算した要約はコンテンツハッシュでキャッシュし、同一サブツリーへの後続クエリは0コストで応答。

インクリメンタル更新

チャンク1つを更新 → O(log n)の祖先ノードをdirtyマーク → 次回クエリ時のみ再計算。RAPTORは全ツリー再構築が必要。

粒度適応クエリ

スコア関数 = cos類似度 + λ×(深さ/H) により、質問のスコープに合わせてツリーの適切な深さのノードを選択。λ>0で葉寄り（細粒度）、λ<0で根寄り（大局的）、λ=0でデフォルト。

def _resolve(node):
    # 葉：そのままテキストを返す（LLM不要）
    if node.is_leaf:
        return node.text

    # キャッシュヒット：0 LLMコール
    if cached := cache.get(node.content_hash):
        return cached

    # ミス：葉テキストから直接1 LLMコールで要約
    leaves = collect_leaves(node)  # 再帰なし
    summary = llm_summarize(leaves)
    cache.set(node.content_hash, summary)
    return summary

def query(question, k=5):
    # Phase 1: プロキシテキストでスコアリング（LLM不要）
    scores = [score(node, question) for node in tree]
    top_k = select_top_k(scores, k)

    # Phase 2: 選択されたk個のみ解決（最大k LLMコール）
    return [_resolve(node) for node in top_k]
